# Apunte de estudio – Tecnologías Disruptivas (VR con Meta SDK y Unity)

El desarrollo de videojuegos para realidad virtual presenta desafíos y necesidades diferentes a los del desarrollo tradicional. Una experiencia de VR no se diseña únicamente para ser observada, sino para ser habitada por el jugador. Por este motivo, gran parte del diseño gira alrededor de la interacción física, la percepción espacial y la sensación de presencia.

Uno de los conceptos fundamentales es el embodiment. Este término hace referencia a la sensación de que el cuerpo virtual pertenece al usuario. Cuando una persona observa manos virtuales que responden correctamente a sus movimientos y acciones, comienza a percibir ese cuerpo digital como una extensión de sí misma. Esta sensación resulta indispensable para generar inmersión.

Relacionado con esto aparece la corporalidad, entendida como la utilización del propio cuerpo como herramienta principal de interacción. En realidad virtual los jugadores giran la cabeza, estiran los brazos, manipulan objetos y se desplazan físicamente dentro del espacio disponible. El diseño debe contemplar estas capacidades y limitaciones para que las acciones resulten naturales.

La memoria muscular también cumple un papel importante. A medida que el jugador repite movimientos, desarrolla hábitos motores que le permiten interactuar de manera más eficiente. Por este motivo los tutoriales deben introducir las mecánicas de forma gradual, permitiendo que el usuario practique acciones básicas antes de enfrentarse a situaciones complejas.

Otro concepto clave es el affordance. Se refiere a la capacidad de un objeto para comunicar visualmente cómo debe utilizarse. Una palanca invita a ser accionada, una manija invita a ser tomada y un botón invita a ser presionado. En VR esto adquiere una importancia todavía mayor porque el usuario espera interactuar con el entorno de forma similar a como lo haría en el mundo real.

Dentro del SDK de Meta, las interacciones se construyen principalmente mediante dos tipos de componentes: los interactors y los interactables. Un interactor representa al elemento que realiza una acción, mientras que un interactable representa al objeto que recibe dicha acción. Las manos, los controladores o los rayos de interacción son ejemplos de interactors. Los objetos agarrables, botones o zonas de teletransporte son ejemplos de interactables.

La mecánica de agarre constituye una de las funcionalidades más utilizadas en VR. Para que un objeto pueda ser tomado normalmente debe poseer componentes específicos que indiquen al sistema que es interactuable. Dependiendo de la configuración elegida, el agarre puede realizarse mediante controladores físicos o mediante hand tracking. El sistema permite además definir cuántos usuarios o manos pueden interactuar simultáneamente con un objeto, lo que resulta útil para herramientas que requieren manipulación con ambas manos.

El hand tracking permite utilizar las manos reales del usuario sin necesidad de controles físicos. Los dispositivos Meta Quest incluyen sensores y cámaras capaces de detectar la posición de los dedos y reconstruir una representación virtual de las manos. Esta tecnología mejora la inmersión, aunque requiere configuraciones específicas dentro del SDK y una correcta implementación de los prefabs correspondientes.

El sistema de interacción visual proporciona información al usuario mediante indicadores conocidos como reticles. Estos elementos permiten comunicar cuándo una superficie es válida para una determinada acción. Son especialmente importantes en sistemas de teletransporte, interacción a distancia y experiencias de realidad mixta donde se realiza análisis de planos o mallas detectadas por el dispositivo.

La locomoción es uno de los aspectos más delicados del diseño en realidad virtual. El movimiento incorrecto puede provocar mareos, desorientación o incomodidad. Por este motivo los sistemas de teletransporte continúan siendo una de las alternativas más utilizadas. Existen implementaciones basadas en áreas de teletransporte y otras basadas en puntos de destino específicos. Ambas permiten desplazar al jugador minimizando el conflicto entre el movimiento visual percibido y la ausencia de movimiento físico real.

Cuando se utilizan desplazamientos continuos es recomendable incorporar ayudas visuales y mecánicas complementarias para reducir el malestar. El diseño de confort constituye una parte fundamental de cualquier proyecto VR exitoso.

El SDK de Meta incorpora una serie de herramientas conocidas como Building Blocks. Estos bloques permiten agregar funcionalidades complejas mediante configuraciones predeterminadas. Gracias a ellos es posible implementar sistemas de interacción, locomoción, seguimiento de manos y otras características sin necesidad de desarrollar cada sistema desde cero. Sin embargo, comprender el funcionamiento interno de cada componente sigue siendo fundamental para resolver errores y adaptar el comportamiento a las necesidades del proyecto.

Los Event Wrappers constituyen otro elemento importante dentro del SDK. Su función consiste en exponer eventos específicos en el Inspector de Unity para que puedan asociarse acciones sin necesidad de escribir código adicional. Gracias a estos componentes es posible reproducir sonidos, activar animaciones, modificar variables, cargar escenas o ejecutar cualquier otro comportamiento cuando ocurre una interacción determinada. Existen wrappers asociados a distintos tipos de interactables y cada uno ofrece eventos específicos relacionados con el objeto que controla.

El sistema Positional Snap permite ubicar objetos en posiciones previamente definidas. Esta herramienta resulta útil para rompecabezas, ensamblaje de piezas, inventarios físicos y cualquier situación donde un objeto deba colocarse en un lugar exacto. La configuración correcta de las físicas y de los componentes de interacción resulta fundamental para evitar comportamientos inesperados al volver a tomar el objeto.

Las interfaces de usuario en VR requieren un tratamiento diferente al de los videojuegos tradicionales. El exceso de elementos en pantalla puede romper la inmersión y generar sobrecarga visual. Siempre que sea posible, la información debe integrarse dentro del propio entorno o asociarse a elementos físicos presentes en la escena. Cuando se utilizan Canvas interactivos es necesario configurar correctamente los sistemas de entrada para garantizar compatibilidad tanto con controladores como con hand tracking.

La optimización adquiere una importancia crítica debido a que los dispositivos de realidad virtual deben mantener tasas de actualización elevadas y estables. Una caída significativa en el rendimiento puede afectar directamente la comodidad del usuario. Para evitar estos problemas se utilizan técnicas como reducción de polígonos, sistemas LOD, occlusion culling, iluminación bakeada y simplificación de materiales. También es importante reducir la cantidad de cálculos realizados por scripts y minimizar el número de objetos activos simultáneamente.

El game feeling en realidad virtual representa la calidad de las sensaciones transmitidas durante la interacción. Una acción debe producir una respuesta clara y coherente mediante animaciones, sonidos, vibraciones, cambios visuales o reacciones físicas. Cuanto más inmediata y consistente sea la respuesta del sistema, más satisfactoria resultará la experiencia para el jugador.

Finalmente, un buen diseño VR debe reflejar tangibilidad del trabajo. Este concepto implica que las acciones del jugador produzcan consecuencias visibles y comprensibles dentro del entorno. El usuario debe percibir que realmente realizó una tarea y que su esfuerzo tuvo impacto sobre el mundo virtual. Armar un mecanismo pieza por pieza, recargar un arma manualmente o colocar objetos en posiciones específicas son ejemplos de actividades que transmiten esta sensación de trabajo tangible y contribuyen significativamente a la inmersión.
